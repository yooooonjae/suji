# 부동산 개발 수지분석 연구 — 설계 문서

날짜: 2026-07-21 · 상태: 사용자 리뷰 대기

## 1. 목표

대한민국 부동산 **개발사업(시행 관점) 수지분석**을 주제로 한 연구 프로젝트.
전국 시장 데이터를 API로 수집·분석하고, 실무 수준의 수지분석 모델을 구현하며,
결과를 **작품 수준의 인터랙티브 웹사이트(아티팩트)** 로 발표한다.

- 관점: 시행사 관점 주거개발(공동주택·오피스텔) 수지분석 중심 + 전국 시장 데이터
- 용도: 실무 검토 가능한 정밀도 + 연구 포트폴리오 완성도, 둘 다
- **타깃 독자(2026-07-21 확정): 시행사·자산운용사·컨설팅** — 기관 보고서 격의 어조·완성도,
  IC(투자심의)/딜 스크리닝에 바로 쓸 수 있는 수준을 지향
- 품질 기준: **G2B(사내 실무도구)와 별개** — 디자인·인터랙션·서사가 완결된 "작품"

### 확장 (2026-07-21 사용자 지시)

- **사업유형 4모드**: 신축분양(기본) + **재개발·재건축·리모델링**(정비사업).
  정비사업 모델: 종전자산평가액·조합원/일반분양 구분·비례율((총수입−총사업비)/종전자산)·
  조합원 분담금(조합원분양가−권리가액)·이주비 이자·명도비, 재개발은 임대주택 의무비율·현금청산,
  리모델링은 기존 세대 유지+증축 일반분양·세대당 분담금 중심.
- **상권분석**: 소상공인시장진흥공단 상가(상권)정보 API(활용신청 필요)로 전국 업소·업종 데이터
  수집 → 시도·주요 상권 단위 집계(업종 구성·밀도) + 개발 유형별 상권 해석.
- **컨설팅 레이어**: 사업유형별 추진 절차(정비사업 단계 포함)·단계별 리스크 체크리스트·
  시장 국면×사업유형 전략 매트릭스를 사이트 콘텐츠로 제공.

## 2. 산출물

1. `~/개발` 연구 코드베이스 (수집→분석→베이크 파이프라인, 테스트 포함)
2. 수지분석 모델 (검증된 계산 코어, Python + 웹 JS 이식)
3. 아티팩트 웹사이트 1편 (단일 HTML, 데이터 임베드, 클라이언트 사이드 계산기)

## 3. 프로젝트 구조

```
~/개발/
├── config.json               # data.go.kr 키(~/.g2b 재사용)·ECOS 키 등
├── data/                     # 수집 원본 캐시 (JSON/SQLite, git 제외)
├── src/
│   ├── collect/              # API 수집기 (소스별 1모듈)
│   ├── analysis/             # 시장 분석·수지 모델
│   └── build/                # out/ JSON 베이크 + site/ 조립
├── out/                      # 베이크된 데이터 (사이트 임베드용)
├── site/                     # 아티팩트 HTML 소스 (섹션·CSS·JS 분리 후 조립)
├── tests/                    # 수지모델 단위테스트 등
└── docs/superpowers/specs/   # 본 문서
```

## 4. 데이터 소스 (전국)

| 소스 | 내용 | 키 | 폴백 |
|---|---|---|---|
| 국토부 실거래가 API (data.go.kr) | 아파트 매매·분양권 실거래 — 시도별 대표 시군구 표본 | 보유(G2B 키, 활용신청 필요시 신청) | 공개 CSV 다운로드 |
| 국토부 건축인허가·주택인허가 통계 | 인허가·착공·준공 물량 추이 | 보유 키 | KOSIS 공표 통계 |
| HUG 민간아파트 분양가격 (data.go.kr) | 시도별 ㎡당 분양가 추이 | 보유 키 | 월간 보도자료 수치 |
| 한국부동산원 R-ONE | 매매가격지수·전세가격지수·미분양 (시도별) | 미보유 → 발급 시도 | KOSIS API 또는 공표 파일 |
| 한국은행 ECOS | 기준금리·대출금리 시계열 | 미보유 → 즉시 발급 | 공표 시계열 수기 |
| 건설공사비지수 (KICT/KOSIS) | 공사비 상승 추이 | KOSIS | 공표 파일 |

원칙: **API 오류·쿼터를 침묵으로 삼키지 않는다**(상태 로그·재시도·명시적 폴백 전환).
수집 원본은 전량 캐시하고, 모든 집계값은 원본 대비 검증 스크립트를 둔다.
전국 커버리지는 시도 단위 통계를 기본으로, 실거래 상세는 시도별 대표 시군구 표본으로 밀도를 확보한다.

## 5. 수지분석 모델 명세

시행 관점 사업수지표. 입력은 실무 수지표 항목을 따른다.

**수입부**: 분양수입 = Σ(타입별 세대수 × 공급면적 × 평당 분양가) × 분양률, 부대수입(상가 등, 옵션)

**지출부**:
- 토지비: 매입비 + 취득세(4.6%) + 중개·명도 등 부대
- 공사비: 연면적 × 평당 공사비 (PCM 모델 계수를 기본값 참조)
- 간접비: 설계·감리·CM비, 인허가 부담금(학교용지·광역교통 등 요율 옵션)
- 판매비: 분양대행·광고비 (분양수입 대비 요율)
- 금융비: 브릿지·PF 이자 — 단순화 모델(대출액 × 금리 × 가중평균 인출기간), 수수료 포함
- 예비비: 총지출 대비 요율

**지표**: 개발이익, 마진율(이익/총수입·이익/총지출), 자기자본수익률,
분기 현금흐름 기반 NPV·IRR, 민감도(분양가·공사비·분양률·금리 ±)

**검증**: 단위테스트(경계값·알려진 사례 대조), Python 코어 ↔ 웹 JS 이식본 결과 일치 테스트.

## 6. 분석·연구 콘텐츠

1. 전국 시장 사이클: 시도별 매매지수·미분양·인허가 추이와 국면 판정
2. 마진 스퀴즈: 공사비지수 vs 분양가 상승률 격차 분석
3. 표준 사례 수지분석 3건 (실데이터 파라미터 기반 가상 사례: 수도권 공동주택, 지방 광역시 공동주택, 오피스텔)
4. 민감도 연구: 사업성을 좌우하는 변수 순위(토네이도)와 손익분기 조건

## 7. 웹사이트 — 새 구성 (작품 기준)

**콘셉트**: "사업수지 보고서 × 건축 도면" — 도면 그리드·치수선 모티프 위에
에디토리얼 타이포그래피로 전개되는 챕터형 단일 페이지. 대시보드가 아니라 **읽는 작품**.

**정보 구조** (스크롤 챕터, 상단 고정 챕터 내비):
- 서장: 히어로 — 주제 선언, 핵심 수치 카운터 애니메이션
- Ⅰ. 시장 — 전국 시도별 지표(지도/스몰멀티플), 사이클 서사
- Ⅱ. 구조 — 수지표 해부(수입·지출 구성 인터랙티브 워터폴)
- Ⅲ. 계산기 — 인스트루먼트 패널: **사업유형 4모드(신축분양·재개발·재건축·리모델링) 전환**,
  입력(유형별 프리셋+슬라이더) → 실시간 수지표·워터폴·IRR/마진 게이지·비례율/분담금(정비사업),
  시나리오 A/B 비교
- Ⅳ. 민감도 — 토네이도 차트, 2-변수 히트맵(분양가×공사비 손익분기)
- Ⅴ. 상권 — 시도·주요 상권 업종 구성/밀도, 개발 유형별 상권 해석
- Ⅵ. 사례 — 표준 사례 카드→상세 (신축 3건 + 재건축 1건)
- Ⅶ. 컨설팅 — 사업유형별 추진 절차 타임라인, 단계별 리스크 체크리스트,
  시장 국면×사업유형 전략 매트릭스
- Ⅷ. 방법론 — 데이터 출처·수집일·모델 가정·한계 명시

**디자인 원칙**: dataviz·artifact-design 스킬 준수, 라이트/다크 완전 대응,
시스템 한글 폰트 스택(외부 폰트 불가 — CSP), 차트는 인라인 SVG 자작,
스크롤 리빌·호버 디테일 등 마이크로 인터랙션, 모바일 대응.
숫자는 라틴 세리프 악센트로 재무 보고서의 격을 살린다.

## 8. 품질 기준

- 수지모델: 단위테스트 통과 + Python↔JS 일치 검증
- 데이터: 수집 로그·검증 스크립트, 출처·기준일을 사이트에 명시
- 웹: 디자인 자체 리뷰(타이포 위계·간격·색 대비·다크모드) 후 배포, 렌더링 실측 확인
- 코드: 커밋 단위 작업, 외부 리뷰(codex) 1회 이상

## 9. 단계 (구현 계획의 뼈대)

1. 셋업: config·키 확인, API 활용신청/발급 (막히는 소스는 폴백 확정)
2. 수집기 구현 + 원본 캐시 (소스별 병렬)
3. 분석: 시장 지표 정리, 마진 스퀴즈, 사례 파라미터 산출
4. 수지모델 코어(Python) + 테스트 → JS 이식 + 일치 검증
5. 베이크: out/ JSON 생성
6. 사이트: 디자인 시스템 → 섹션 구현 → 계산기 → 차트 → 인터랙션
7. 검수: 렌더링 확인, 디자인 리뷰, codex 교차 리뷰 → 아티팩트 배포
